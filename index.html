git init: para iniciar el repo vim index.html git add ./*: creo archivos luego los agrego al control de git git commit -m "primer commit": cierro "la cajita" con ese archivo y los cambios efectuados. git remote add origin https://github.com/rodmontiel/acamica.git: sincronizo mi repo local con el remoto. git status: verifico estado de git git push -u origin master (o solamente git push): pusheo mi repo local al remoto git pull -u origin master (o simplemente git pull): me traigo los cambios del repo remoto a mi repo local asi quedamos sincronizados. git clone https://blabla/acamica.git: clono un repo remoto localmente. git branch: lista los branches. git branch nuevo_branch: crea un nuevo branch/rama. git checkout nuevo_branch y git add./ y git commit agregaria archivos a tracear y commitearia cambios en el branch nuevo_branch. git merge nuevo_branch: mergearia cambios a master. Si hay conflictos, me dice cual es el archivo y me escribe alli las diferencias. Desde alli borro y luego ejecuto un commit para resolver conflictos. git merge nuevo_branch: finalmente hago el merge de la rama, que ya cuenta con una version commiteada con el conflicto resuelto.
